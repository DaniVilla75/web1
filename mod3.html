Introducción a HTML

HTML (HyperText Markup Language) es el lenguaje fundamental para crear páginas web. Es un lenguaje de marcado que define la estructura y el contenido de una página web.

1. Conceptos básicos

Los orígenes de HTML (1989-1991)

  • Creado por Tim Berners-Lee en el CERN.
  • 1989: propuso un sistema de hipertexto basado en Internet.
  • 1990: desarrolló el navegador/editor WorldWideWeb.
  • 1991: publicó "HTML Tags".

HTML 2.0 (1995)

  • Primera versión estandarizada por el IETF.
  • Soportaba formularios simples.

HTML 3.2 (1997)

  • Desarrollado por el W3C.
  • Incluía tablas, applets, texto fluido con imágenes.

HTML 4.01 (1999)

  • Introdujo separación entre estructura y presentación.
  • Tres variantes: Strict, Transitional, Frameset.

XHTML (2000)

  • Basado en XML, más estricto.
  • Menos tolerante a errores, en desuso.

HTML5 (2014 - Actual estándar)

  • Nuevos elementos semánticos: <header>, <footer>, <section>, etc.
  • Soporte para video/audio, canvas, almacenamiento local.
  • Mejoras en formularios, compatibilidad móvil y accesibilidad.

HTML Living Standard

  • Desarrollo continuo sin versiones fijas, mantenido por WHATWG.

Futuro de HTML

  • Web Components, APIs modernas, PWAs, accesibilidad mejorada.

2. Estructura básica de un documento HTML

<!DOCTYPE html>
          <html lang="es">
          <head>
          <meta charset="UTF-8">
          <meta name="viewport" content="width=device-width, initial-scale=1.0">
          <title>Título de la página</title>
          </head>
          <body>
          <!-- Contenido de la página -->
          </body>
        </html>

Explicación de:

  • <!DOCTYPE html>: tipo de documento
  • <html>: contenedor principal
  • <head>: metadatos
  • <body>: contenido visible

4. Atributos HTML

¿Qué son los atributos? Son modificadores que agregan información o funcionalidad a las etiquetas HTML. Se escriben dentro de la etiqueta de apertura como nombre="valor".

Atributos Globales
  • id: Identificador único de un elemento. <div id="principal">
  • class: Asigna clases a un elemento. <p class="texto">
  • style: Estilo en línea. <h1 style="color:red">
  • title: Texto emergente al pasar el mouse. <img title="Logo">
Atributos Específicos
  • href: Enlace en <a>. <a href="https://...">
  • src: Ruta de imagen o script. <img src="imagen.jpg">
  • alt: Texto alternativo para imágenes. <img alt="Descripción">
Atributos Booleanos
  • disabled: Desactiva el elemento. <button disabled>
  • readonly: Solo lectura. <input readonly>
  • required: Campo obligatorio. <input required>

Estos atributos permiten personalizar y controlar el comportamiento y estilo de cada elemento HTML.

5. Estructura Semántica

Es como la cabeza de tu página. Aquí va el logo, el título y a veces el menú de navegación.
Ejemplo: El nombre de tu página y el menú principal.

Es la barra de navegación con enlaces que te llevan a otras partes.
Ejemplo: Links de "Inicio", "Productos", "Contacto".

Contenido principal de la página. Debe haber solo uno.
Ejemplo: Lista de videos en YouTube, productos en una tienda.

Como un artículo de periódico o post de blog que tiene sentido por sí solo.
Ejemplo: Tweet, entrada de blog, comentario en un foro.

Divide el contenido en secciones lógicas como capítulos de un libro.
Ejemplo: "Últimas noticias", "Testimonios".

Contenido relacionado pero no esencial.
Ejemplo: Barra lateral de blog con anuncios o artículos sugeridos.

Parte inferior de la página.
Ejemplo: "© 2024 MiWeb - Síguenos en Instagram".

<figure> envuelve una imagen o gráfico, y <figcaption> es el pie de foto.
Ejemplo: 
<figure>
                  <img src="perro.jpg" alt="Un perro jugando">
                  <figcaption>Mi perro Loki en el parque</figcaption>
                </figure>
¿Por qué usar esto?
  • Mejor SEO: los buscadores entienden mejor tu contenido.
  • Más accesibilidad: los lectores de pantalla interpretan mejor la estructura.
  • Código más limpio y entendible.

6. Formularios

Los formularios HTML son herramientas esenciales para recolectar información de los usuarios en una página web. A continuación, te explico cada elemento con definiciones sencillas y ejemplos prácticos.

<form> con atributos action y method

Definición: La etiqueta <form> define un formulario. Los atributos más importantes son:

  • action: URL donde se enviarán los datos.
  • method: Método de envío (GET o POST).

Ejemplo:

<form action="/procesar-datos" method="POST"> <! Aquí los campos del formulario > </form>

📌 En este caso, cuando el usuario envíe el formulario, los datos irán a /procesar-datos usando el método POST.

Campos <input>

Definición: permite crear diferentes tipos de campos para que el usuario ingrese datos. Algunos tipos comunes:

  • text: Texto normal
  • password: Contraseña oculta
  • email: Correo electrónico
  • number: Solo números

Ejemplo:

<input type="text" placeholder="Nombre"> <input type="password" placeholder="Contraseña"> <input type="email" placeholder="Correo electrónico">

🔹 El atributo placeholder muestra un texto de ayuda dentro del campo.

<textarea>

Definición: Para ingresar textos largos.

<textarea rows="4" cols="50" placeholder="Escribe tu mensaje aquí..."></textarea>

Ejemplo:

<textarea rows="4" cols="50" placeholder="Escribe tu mensaje aquí..."></textarea>

📏 rows y cols definen el tamaño del área de texto.

Desplegables: <select> y <option>

Definición: Permiten elegir una opción predefinida.

Ejemplo:

<select> <option value="mx">México</option> <option value="es">España</option> <option value="ar">Argentina</option> </select>

🌎 El usuario selecciona una opción y se envía el value correspondiente.

Botones: <button> y <input type="submit">

Definición: Permiten enviar el formulario o ejecutar acciones.

  • button:
  • input type="submit" envía el formulario.

Ejemplo:

<button type="button">Solo un botón</button> <input type="submit" value="Enviar">

🚀 El botón de submit procesa el formulario cuando se hace clic.

Validación: required, pattern, min, max ✅

Definición: atributos que ayudan a validar datos antes de enviarlos:

  • required: obliga a llenar el campo.
  • pattern: define una expresión regular para validar (ej: formato de teléfono).
  • min/max: establece límites numéricos.

Ejemplo:

<input type="text" required placeholder="Nombre (obligatorio)"> <input type="number" min="18" max="99" placeholder="Edad"> <input type="tel" pattern="[0-9]{10}" placeholder="Teléfono (10 dígitos)">

✅ Si el usuario no cumple las reglas, el formulario no se envía.

<label> para accesibilidad

Definición: Asocia un texto descriptivo a un campo del formulario, mejorando la accesibilidad.

Ejemplo:

<label for="nombre">Nombre completo:</label> <input type="text" id="nombre">

👉 Haciendo clic en el texto "Nombre completo", el cursor se posiciona en el campo.

Agrupación con <fieldset> y <legend>

Definición: Organizan los formularios largos en bloques lógicos.

  • fieldset>: agrupa elementos relacionados.
  • legend>: añade un título al grupo.

Ejemplo:

<fieldset> <legend>Información personal</legend> <label for="nombre">Nombre:</label> <input type="text" id="nombre"> <label for="edad">Edad:</label> <input type="number" id="edad"> </fieldset>

📌 Útil para organizar formularios largos en secciones.

🌟 Ejemplo completo de formulario
<form action="/registro" method="POST"> <fieldset> <legend>Registro de usuario</legend> <label for="usuario">Usuario:</label> <input type="text" id="usuario" required> <label for="clave">Contraseña:</label> <input type="password" id="clave" required> <label for="pais">País:</label> <select id="pais"> <option value="mx">México</option> <option value="es">España</option> </select> <label for="comentarios">Comentarios:</label> <textarea id="comentarios"></textarea> <input type="submit" value="Registrarse"> </fieldset> </form>

7. Tablas

Las tablas permiten mostrar información en filas y columnas. HTML ofrece etiquetas específicas para construir estructuras organizadas, desde datos simples hasta informes complejos.

📌 Estructura básica

Las tablas en HTML se crean con la etiqueta table. Dentro de ella, cada fila se define con tr (table row), y cada celda con td (table data). Para celdas de encabezado usamos th (table header).

NombreEdadCiudad
María25Madrid
Carlos30Barcelona
<table> <tr> <th>Nombre</th> <th>Edad</th> <th>Ciudad</th> </tr> <tr> <td>María</td> <td>25</td> <td>Madrid</td> </tr> <tr> <td>Carlos</td> <td>30</td> <td>Barcelona</td> </tr> </table>
📌 Secciones de tabla: thead, tbody, tfoot

Estas etiquetas ayudan a organizar la tabla en secciones lógicas:

  • thead: contiene los encabezados de la tabla
  • tbody: contiene el cuerpo principal de datos
  • tfoot: contiene el pie de tabla (totales, resúmenes)
ProductoPrecio
Laptop€899
Teléfono€599
Total€1498
<table> <thead> <tr> <th>Producto</th> <th>Precio</th> </tr> </thead> <tbody> <tr><td>Laptop</td><td>€899</td></tr> <tr><td>Teléfono</td><td>€599</td></tr> </tbody> <tfoot> <tr><th>Total</th><td>€1498</td></tr> </tfoot> </table>

Explicación: Esto ayuda a los navegadores y motores de búsqueda a entender mejor la estructura de tu tabla.

📌 Combinar celdas: rowspan y colspan

Permiten que una celda ocupe varias columnas o filas:

  • colspan: combina celdas horizontalmente (columnas).
  • rowspan: Combina celdas verticalmente (filas).
ID Notas
Primer Parcial Segundo Parcial
001 85 90
<table> <tr> <th rowspan="2">ID</th> <th colspan="2">Notas</th> </tr> <tr> <th>Primer Parcial</th> <th>Segundo Parcial</th> </tr> <tr> <td>001</td> <td>85</td> <td>90</td> </tr> </table>

8. Multimedia e Incrustación

HTML permite enriquecer las páginas con contenido multimedia como imágenes, audio, video, gráficos vectoriales y contenidos incrustados.

📷 1. Imágenes: <img> y formatos recomendados

<img> se usa para insertar imágenes. Requiere al menos el atributo src que especifica la ruta de la imagen y el alternativo alt.

Formatos: JPEG (fotos), PNG (transparencia), WebP (óptimo), SVG (vectorial).

<img src="foto-perfil.jpg" alt="Foto de perfil del usuario" width="300" height="200">
🗺️ 2. Mapas de imagen: <map> y <area>

Permiten definir zonas clickeables dentro de una imagen. Se usa junto con img añadiendo el atributo usemap.

<img src="mapa-mundo.png" alt="Mapa del mundo" usemap="#mapamundo"> <map name="mapamundo"> <area shape="rect" coords="34,44,270,350" alt="América" href="america.html"> <area shape="circle" coords="337,300,44" alt="África" href="africa.html"> <area shape="poly" coords="100,100,200,50,300,100,350,300" alt="Europa" href="europa.html"> </map>
🎵 3. Audio: <audio>

Estos elementos permiten incrustar contenido multimedia directamente en la página sin necesidad de plugins.

<audio controls> <source src="musica.mp3" type="audio/mpeg"> <source src="musica.ogg" type="audio/ogg"> Tu navegador no soporta el elemento de audio. </audio>
🎥 4. Video: <video>
<video width="320" height="240" controls poster="preview.jpg"> <source src="video.mp4" type="video/mp4"> <source src="video.webm" type="video/webm"> Tu navegador no soporta el elemento de video. </video>
🌐 5. Contenido externo: <iframe>

Permite incrustar otro documento HTML dentro de la página actual. Se usa comúnmente para videos de YouTube, mapas de Google, etc.

<iframe src="https://www.youtube.com/embed/dQw4w9WgXcQ" width="560" height="315" frameborder="0" allowfullscreen> </iframe>
✏️ 6. Gráficos vectoriales: <svg>

SVG (Scalable Vector Graphics) es un formato de imagen vectorial basado en XML que puede ser manipulado con CSS y JavaScript.

<svg width="100" height="100"> <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" /> <text x="50" y="55" text-anchor="middle" fill="black">Hola</text> </svg>
<svg width="200" height="200"> <rect x="50" y="50" width="100" height="100" fill="blue" /> <polygon points="100,10 40,198 190,78 10,78 160,198" fill="red" /> </svg>

9. Metadatos y SEO Básico 🔍

Los metadatos ayudan a describir el contenido de una página web. El SEO (Search Engine Optimization) mejora su visibilidad en los resultados de búsqueda.

1. <title>: Título de la página

El título aparece en la pestaña del navegador y es fundamental para el SEO.

<title>Guía de HTML5 - Aprende a crear páginas web</title>
  • ✔ Entre 50-60 caracteres.
  • ✔ Incluir palabras clave.
  • ✔ Atractivo para el usuario.
2. Metaetiquetas <meta>

a) <meta name="description">: Resumen que aparece en Google.

<meta name="description" content="Aprende HTML5 desde cero con ejemplos prácticos. Tutorial paso a paso para principiantes.">
  • ✔ Máximo 160 caracteres.
  • ✔ Incluir palabras clave.
  • ✔ Redacción atractiva para generar clics.

b) <meta name="keywords">: Palabras clave (hoy en desuso).

<meta name="keywords" content="HTML, CSS, JavaScript, tutorial web, programación">
3. Favicon

Es el ícono que aparece en la pestaña del navegador.

<link rel="icon" href="favicon.ico" type="image/x-icon">

Formatos: .ico, .png, .svg (16x16px o 32x32px)

4. Open Graph para redes sociales

Estas metaetiquetas permiten personalizar cómo se muestra tu página al compartirla en redes.

<meta property="og:title" content="Aprende HTML5 en 2024"> <meta property="og:description" content="Curso completo de HTML5 para principiantes"> <meta property="og:image" content="https://ejemplo.com/imagen.jpg"> <meta property="og:url" content="https://ejemplo.com">
5. Estructura Semántica y SEO

Usar etiquetas semánticas mejora el SEO y la accesibilidad:

  • <header> – Cabecera
  • <nav> – Menú
  • <main> – Contenido principal
  • <article> – Artículo o post
  • <section> – Agrupación temática
  • <aside> – Contenido complementario
  • <footer> – Pie de página

Ejemplo completo:

<body> <header> <h1>Mi Blog de Tecnología</h1> <nav> <a href="/">Inicio</a> <a href="/tutoriales">Tutoriales</a> </nav> </header> <main> <article> <h2>Cómo aprender HTML5</h2> <p>Guía completa para principiantes...</p> </article> </main> <footer> <p>© 2024 Mi Blog</p> </footer> </body>
📌 Buenas Prácticas SEO
  • ✅ Títulos claros y descriptivos
  • ✅ Descripciones concisas y llamativas
  • ✅ Semántica con HTML5
  • ✅ Alt en imágenes
  • ✅ Enlaces internos/externos
  • ✅ Velocidad de carga optimizada

💡 Aplicar estos elementos mejora la visibilidad de tu web en Google y en redes sociales. ¡Hacelo parte de tu rutina al crear páginas!

10. Buenas Prácticas y Accesibilidad Web ♿💡

La accesibilidad web garantiza que todas las personas, incluyendo quienes tienen discapacidades, puedan navegar y entender el contenido. Estas prácticas siguen las recomendaciones WCAG.

1. HTML válido y bien formado

Un documento HTML debe seguir reglas de sintaxis: etiquetas cerradas, atributos bien escritos y estructura jerárquica coherente.

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <title>Página Accesible</title> </head> <body> <h1>Bienvenido</h1> <p>Este es un párrafo correctamente cerrado.</p> </body> </html>

Errores comunes:

  • ❌ Etiquetas sin cerrar
  • ❌ Atributos mal escritos (ej: <img alt> sin comillas)

📎 Usá el Validador W3C para comprobar tu HTML.

2. Uso adecuado de etiquetas semánticas

Las etiquetas semánticas <header>, <main>, <nav>, etc., ayudan a estructurar el contenido y mejoran la accesibilidad.

<header> <h1>Noticias del día</h1> <nav> <ul> <li><a href="/">Inicio</a></li> <li><a href="/contacto">Contacto</a></li> </ul> </nav> </header> <main> <article> <h2>Título de la noticia</h2> <p>Contenido importante...</p> </article> </main>
  • ❌ No usar <div> para todo.
  • ❌ No saltarse niveles de encabezado (<h1> seguido de <h3>).
3. Atributos ARIA

Los atributos ARIA mejoran la accesibilidad en interfaces dinámicas.

<button aria-label="Cerrar ventana">X</button>
<div class="decoración" aria-hidden="true"></div>

También se usa role para definir tipos de elementos (role="navigation", role="alert"...).

4. Texto alternativo en imágenes

El atributo alt permite describir imágenes a lectores de pantalla o si no se cargan.

<img src="grafico.jpg" alt="Gráfico de ventas 2024: aumento del 20%">
<img src="divisor.jpg" alt="">

Errores comunes: alt="imagen", o directamente no usar alt.

5. Contraste adecuado (CSS)

El texto debe tener suficiente contraste con el fondo. La relación recomendada es al menos 4.5:1.

body { background: white; color: black; /* Contraste 21:1 */ }
body { background: lightgray; color: white; /* Contraste bajo */ }

🎨 Usá WebAIM Contrast Checker para validar tus combinaciones.

6. Navegación con teclado

El sitio debe poder usarse con teclado (Tab, Enter, flechas) para quienes no usan mouse.

Prácticas clave:

  • ✔ Usar tabindex="0" para elementos enfocados
  • ✔ Usar tabindex="-1" para saltar foco
  • ✔ Estilos claros para :focus
  • ❌ No usar tabindex mayor a 0
  • Inicio
  • Contacto
button:focus { outline: 2px solid blue; }
📌 Resumen de buenas prácticas
  • ✅ HTML válido
  • ✅ Etiquetas semánticas
  • ✅ Atributos ARIA donde haga falta
  • ✅ Texto alternativo en imágenes
  • ✅ Contraste legible
  • ✅ Navegación accesible con teclado

Aplicar estos principios mejora la experiencia para todos los usuarios. ¡Un web accesible es un web más inclusivo! 🌍

11. Integración con otros lenguajes 🔗

HTML no trabaja solo: se puede combinar con otros lenguajes como CSS, JavaScript o lenguajes de servidor como PHP, Python o Node.js para lograr sitios más atractivos y funcionales.

1. Relación con CSS y JavaScript

CSS: define los estilos visuales de la página.

<link rel="stylesheet" href="estilos.css">

JavaScript: agrega interactividad (validaciones, animaciones, eventos).

<script src="script.js"></script>
2. Atributo style (uso puntual)

El atributo style permite aplicar estilos directamente en una etiqueta HTML. Es útil para pruebas rápidas o estilos puntuales.

<p style="color: blue; font-size: 16px;">Texto azul</p>

⚠️ Recomendación: Para proyectos grandes, usá siempre CSS externo.

3. Eventos básicos en HTML

Los eventos permiten ejecutar código cuando ocurre una acción (clic, carga, teclado, etc.).

Ejemplo con onclick:

<button onclick="alert('Hola')">Click</button>

Ejemplo con onload:

<body onload="console.log('Página cargada')">

✨ Estos conceptos permiten construir páginas dinámicas, personalizables y mucho más atractivas.
